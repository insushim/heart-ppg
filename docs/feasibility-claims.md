# 스마트폰 카메라 PPG 건강측정 앱 — 기술 실현가능성 주장 (검증 대상)

검증 요청: 아래 각 주장이 **과학적으로 타당한지 / 과대광고인지**, 스마트폰(안드로이드, IR센서 없음) 환경 기준으로 독립 평가해줘.
특히 "측정 가능하다"고 적힌 것 중 실제로는 신뢰불가/사이비인 항목을 색출하고, 정확도 수치(±오차, 상관계수)가 현실적인지 봐줘.

## A. 접촉식 손가락 PPG (카메라+플래시에 손가락 대기)
- A1. **심박수(HR)**: red 채널 평균 시계열 → 밴드패스(0.7~4Hz) → FFT/피크검출 → BPM. ECG 대비 ±2~3 bpm, r>0.95. → 신뢰 가능하다고 주장.
- A2. **HRV(심박변이도)**: 비트간 간격(IBI/RR) → RMSSD, SDNN, pNN50, LF/HF. 60~120초 측정 시 RMSSD는 단기 지표로 쓸만함. 단 카메라 PPG는 ECG R-peak보다 jitter 커서 절대값 신뢰도는 떨어짐(상대추세용). → 부분 신뢰.
- A3. **스트레스 지수**: HRV 기반(Baevsky Stress Index 또는 LF/HF, RMSSD 역수 매핑). 정성적 추세는 가능하나 절대 점수는 캘리브레이션 필요. → 추세용으로만 신뢰.
- A4. **호흡수(RR)**: 호흡성 동성부정맥(RSA)으로 PPG 진폭/주파수 변조 → 0.1~0.5Hz 성분 추출. 안정 시 ±2~3 bpm. → 조건부 신뢰.
- A5. **SpO2(산소포화도)**: 폰 카메라는 IR 없음. red/blue(또는 red/green) ratio-of-ratios 시도하나 캘리브레이션 불가, 임상 신뢰도 없음. → **과대광고. 제공 금지 또는 강한 면책 필요**라고 주장.
- A6. **혈압(BP)**: 단일 손가락 PPG만으로 PTT(맥파전달시간) 측정 불가. 커프 캘리브레이션 없이는 신뢰불가. → **사이비 수준. 제공 금지**라고 주장.

## B. 비접촉 얼굴 rPPG (카메라로 얼굴 영상)
- B1. **얼굴 rPPG로 HR**: CHROM, POS(Plane-Orthogonal-to-Skin), ICA, 딥러닝(DeepPhys 등)으로 피부 미세 색변화에서 맥박 추출. green 채널 주력. 안정 조명·정지 시 ±3~5 bpm. 손가락보다 SNR 낮고 움직임/조명에 취약. → 조건부 신뢰, 손가락 대비 정확도 낮음.
- B2. **얼굴 혈색/안색 추가가 손가락 PPG 정확도를 높이나?**: 핵심 질문. 내 판단 = HR 정확도 자체는 손가락이 이미 우수하므로 얼굴이 "정확도를 올린다"는 건 오해. 다만 (a)센서퓨전으로 robustness/이상치 제거, (b)비접촉 편의성, (c)안색 기반 정성 지표(혈색 톤, 좌우대칭, 추세)는 별개 modality로 가치 있음. → "정확도↑"가 아니라 "modality 다양성·편의성↑"로 포지셔닝해야 함.

## C. 규제/법적
- C1. 이 모든 수치는 **웰니스/참고용**이며 의료기기 아님. 진단·치료 주장 금지. Google Play 건강앱 정책·의료기기 규제(국내 식약처/해외 FDA) 회피 위해 면책·비진단 문구 필수. → 타당하다고 주장.
