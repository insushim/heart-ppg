# 심박·건강 측정 앱 개발계획 (Android, 카메라 PPG)

> 4-way 독립검증(로컬 qwen · Gemini · Opus · codex GPT-5.3-spark) **전 계열 합의** 기반.
> 핵심 원칙: **측정 가능한 것만 정직하게. SpO2·혈압은 제공하지 않는다.**
>
> ⚠️ GPT 계열 추가 지적(반영됨):
> - 정확도 절대수치(±2~3bpm, r>0.95) **광고 금지** — 실험실 조건값, 표시광고법 위반 소지
> - **"스트레스 지수" 명칭 폐기** → "자율신경 활성도 / 컨디션 밸런스" + 신호등
> - HR/HRV도 "의료 목적" 문구 붙으면 규제 대상 격상 / 기기·OS·조명별 편차 면책 필요

## 1. 제품 정의
- **이름(가칭)**: Heart — 카메라 심박·컨디션 측정
- **포지셔닝**: 의료기기 아님. **웰니스/참고용** 셀프 모니터링.
- **타깃**: 손가락을 카메라+플래시에 대고 30~60초 → 심박/HRV/스트레스/호흡/관류 + 일별 컨디션 추세.
- **차별점**: (a)정직한 지표만 (b)일별 추세·회복도 (c)선택적 얼굴 비접촉 측정.

## 3. 핵심 신호처리 파이프라인 (손가락 PPG)
```
CameraX 프레임 → 손가락 감지(빨강 우세 + 밝기) → ROI 평균 RGB
 → red 채널 시계열(>=30fps) → detrend → 밴드패스 0.7~4Hz
 → 신호품질 게이트(SNR/PI) → 피크검출 + FFT
 → HR(bpm) / IBI배열 → HRV(RMSSD,SDNN,pNN50)
 → 스트레스 매핑(신호등) / RSA → 호흡수 / AC·DC → PI
```
**필수 구현 디테일 (검증에서 지적된 함정):**
- **노출/포커스/화이트밸런스 고정(AE/AF/AWB lock)** — 안 하면 카메라 자동보정이 신호를 왜곡
- **플래시 발열·안전**: 측정시간 상한(최대 60s), 광과민성 경고
- **신호품질 게이트**: SNR/PI 미달 시 "손가락을 잘 밀착하세요" 재시도 (쓰레기 측정 차단)
- **피부톤 편향 대응**: Fitzpatrick IV~VI 테스트, ROI 적응형 게인
- **워밍업 버림**: 첫 2~3초 데이터 폐기(AE 안정화 전)

## 4. 기술 스택
- **언어/플랫폼**: Kotlin + Jetpack Compose, minSDK 26+
- **카메라**: CameraX (ImageAnalysis use-case, YUV→RGB)
- **신호처리**: Kotlin 자체구현(밴드패스·FFT·피크검출) 또는 JNI/C++(성능). FFT는 별도 경량 라이브러리
- **얼굴 rPPG(v2)**: ML Kit Face Detection으로 ROI(이마/뺨) → CHROM/POS 알고리즘 → (옵션)TFLite 딥러닝 모델
- **로컬 저장**: Room(SQLite) — 측정 이력·추세
- **차트**: Vico 또는 MPAndroidChart
- **백엔드(선택)**: 초기엔 로컬 온리(프라이버시·규제 단순). 클라우드 동기화는 후순위
- **테스트**: 합성 PPG 신호로 알고리즘 단위테스트(정답 BPM 주입 → 오차 검증)

## 5. 개발 단계
**Phase 0 — 신호처리 코어 검증 (1~2주)**
- CameraX 프레임 → red 시계열 추출 PoC
- 밴드패스+FFT+피크검출으로 HR 산출, **시중 손가락 산소포화도계/스마트워치와 ±3bpm 일치 검증**
- 합성신호 단위테스트 셋업

**Phase 1 — MVP (HR + HRV + 스트레스 신호등) (2~3주)**
- 측정 UI(손가락 가이드, 진행바, 실시간 파형)
- 신호품질 게이트·AE lock
- Room 저장 + 결과 화면 + 면책문구

**Phase 2 — v1.1/1.2 (호흡·PI·추세·리듬플래그) (2~3주)**
- 호흡수(3조건 게이트), 관류지수, 일/주 추세 그래프
- 불규칙 리듬 플래그(false positive 처리 로직 포함)

**Phase 3 — v2.0 (얼굴 rPPG 비접촉 + 안색) (3~4주)**
- ML Kit ROI + CHROM/POS
- **센서퓨전: 신호품질(SNR) 가중 — 단순평균 금지(역오염 방지)**
- 안색 추세(실험 라벨)

## 6. 규제·법적 (검증 필수사항)
- **측정 화면 내 직관 노출**: "본 앱은 질병의 진단·치료·예방 목적으로 사용할 수 없습니다" (약관 깊이 숨기면 무효)
- SpO2·혈압 **표시 자체 금지** (면책으로도 의료기기 재분류 리스크 못 막음)
- **정확도 절대수치 광고 금지** (±2~3bpm, r>0.95 등) — 실험실 조건값, 표시광고법 위반 소지 (GPT 지적)
- **HR/HRV도 "의료 목적" 표현 금지** — 붙는 순간 MFDS/FDA/CE MDR 규제 대상 격상 (GPT 지적)
- **기기·OS·조명 편차 면책**: "검증된 기기/환경에서 동작" 조항 (GPT 지적)
- Google Play 건강앱 정책: 스크린샷·설명에서 의료효능 암시 금지
- 한국 식약처 디지털헬스 가이드라인(2023) 준수
- 개인정보: 측정데이터 로컬 우선, 클라우드 전송 시 동의·암호화
