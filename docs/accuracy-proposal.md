# 심박 정확도/상세도 개선안 (검증 요청)

현재: 30초 측정 → 30fps 리샘플 → detrend → 밴드패스(0.7~4Hz) → 단일 FFT(Hann, 4x zero-pad, parabolic) HR + 피크기반 HR, 0.6/0.4 가중. PI/품질 게이트 있음.

아래 개선안 각각의 (a)정확도 기여 타당성 (b)구현위험 (c)스마트폰 손가락 PPG에 적합한지 평가하고, 빠진 더 좋은 방법이 있으면 제시해줘.

## P1. Welch PSD (구간 평균 주기도)
단일 FFT 대신 신호를 겹치는 구간(예: 8~10초, 50% overlap, Hann)으로 나눠 각 주기도를 평균 → 스펙트럼 분산↓ → 잡음 하에서 HR 피크가 더 안정. HR이 측정 중 약간 변해도 dominant 주파수 추정이 견고.
- 기대: 잡음/약한 신호에서 HR 오차↓. 단위테스트로 검증 가능.

## P2. 슬라이딩 윈도우 구간별 HR + 강건 결합 (모션 제거)
전체 30초를 겹치는 윈도우(예: 10초, 2초 step)로 나눠 윈도우별 HR+품질(SNR) 계산 → 저품질/이상치 윈도우 버리고 나머지의 중앙값(median) 채택. 손가락 흔들림/압력변화로 오염된 구간을 자동 배제 → 안정 구간만 사용.
- 기대: 모션 아티팩트에 강해져 정확도↑. 단위테스트 가능(중간에 오염 주입).

## P3. 카메라 프레임레이트 고정 (AE_TARGET_FPS_RANGE)
현재 fps 미지정 → 토치+어두운 장면서 카메라가 15fps 이하로 떨어질 수 있어 시간 해상도↓. Camera2Interop로 30fps(또는 가용 최고)로 고정 → IBI/피크 시간정밀도↑, 리샘플 손실↓.
- 기대: 표본화 일관성↑. 앱단(단위테스트 불가), 위험 낮음.

## P4. 실시간 러닝 BPM 표시 (상세도)
측정 중 ~5초마다 현재까지 데이터로 잠정 HR을 계산해 화면에 실시간 표시. "자세하게" 요구 충족 + 사용자가 안정됐는지 피드백.
- 기대: UX 상세도. 정확도엔 중립.

## P5. 품질 미달 시 적응형 연장
30초 측정 후 품질이 POOR이면 자동으로 최대 45~60초까지 연장 측정 → 더 많은 비트로 추정.
- 기대: 약신호 사용자 구제.

## 질문
- P1~P5 중 정확도 실효가 큰 것 우선순위? 
- 손가락 접촉 PPG에서 red 채널 단독 vs (red+green) 조합이 의미 있나? (토치 사용 시)
- 빠진 더 효과적인 기법(예: 적응형 임계 피크검출, 칼만/추적, 호흡/저주파 제거 강화)?
- 과한 것(over-engineering)이거나 폰 환경서 역효과인 것은?
